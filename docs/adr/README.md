# Architecture Decision Records (ADRs)

Short, dated records of significant, hard-to-reverse decisions for Ember. Lightweight format:
**Status · Context · Decision · Consequences · Alternatives considered.**

| ADR | Title | Status |
|---|---|---|
| [0001](./0001-healthkit-role.md) | Apple HealthKit is an ambient, trailing witness — not a live signal | Accepted |
| [0002](./0002-ingestion-api-and-cli.md) | One normalized ingestion API with many producers (provider, UI, CLI, admin batch) | Accepted |

New ADRs are numbered sequentially and never edited in place once Accepted — supersede with a new ADR instead.
