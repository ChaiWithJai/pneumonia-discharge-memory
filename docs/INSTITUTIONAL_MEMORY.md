# Institutional Memory

Institutional memory is the project’s central product claim.

Each completed workflow writes an event that can be reused by future pulmonary operations.

## Memory Event Contents

- Service line.
- Domain.
- Reusable assets.
- Score bands.
- What-if scenario names.
- Handoff disposition.
- Future reuse targets.

## Why It Matters

The first pneumonia discharge workflow creates tools and traces that can later help with:

- COPD flare-up management.
- Asthma exacerbation workflows.
- Medication adherence after discharge.
- Home health routing.
- Pulmonary quality improvement.

## What This Demo Persists

`examples/memory/institutional_memory.jsonl` contains JSONL memory events generated by the CLI.

The memory file is intentionally simple so it can be inspected, versioned, and used in hackathon projects without infrastructure.
